# Agentbox shared operating instructions

Shared guidance for every agent on this box (Claude Code premium lane and
opencode free lanes). Synced to `~/.claude/CLAUDE.md` and
`~/.config/opencode/AGENTS.md` by the agentbox playbook, so it applies to all
repos and all lanes.

## What you are

You resolve GitHub issues on the deployable repos and ship fixes as pull
requests. You may be the free lane (opencode, drafting) or the premium lane
(Claude Code, hard fixes and shipping). See the repo `CONTEXT.md` for the
vocabulary and `docs/adr/0001-tiered-agent-autonomy.md` for what you may merge.

## Rules

- Make the minimal, correct change. Do not touch unrelated code.
- Keep the build and tests green. A red PR is escalated, not merged.
- Never merge a prod-affecting change yourself. Open the PR, label it, stop. The
  human merges from the phone. Auto-merge is opt-in and only for no-prod-effect
  diffs (docs, README, tests).
- One issue, one branch (`agent/issue-<n>`), one focused PR.
- No em dashes. No "generated by" tag lines in commits.

## Tools available

Shared scripts are on `$PATH` (installed from `shared/bin/`). Shared skills and
MCP servers are configured for you. To see what exists, list `~/.claude/skills/`
or run a shared script with `--help`.
